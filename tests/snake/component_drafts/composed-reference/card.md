# Card

## Description
Card is a grouped surface component used to package related content and actions into a single scannable unit. Use it for previews, summaries, or modular content blocks, and avoid forcing all content into cards when a simpler flowing layout would be clearer.

## Purpose
The purpose of Card is to group related content and actions into a clear, self-contained unit.

## Visual Structure
Card is a bounded container with internal spacing, optional header area, body content, and optional footer actions. It may include media, icons, metadata, or links, but the hierarchy should remain clear with one dominant entry point and a balanced reading order.

## Subcomponents
- Heading
- Text
- Paragraph
- Button
- Link
- Optional Icon
- Optional media area

## States & Variants
- States: default, hover, focus-within, selected, disabled.
- Variants: informational, interactive, summary, feature, compact.
- Layout options: vertical, horizontal, media-first, content-first.

## Properties
- `title`: primary card heading.
- `body`: supporting content.
- `actions`: optional links or buttons.
- `media`: optional image or illustration slot.
- `icon`: optional symbolic support element.
- `variant`: card purpose and emphasis.
- `interactive`: whether the whole card acts as a target.
- `padding`: compact or standard internal spacing.

## Brand Reference
No formal brand specification is required for this draft. Card should use clean surfaces, clear hierarchy, and restrained decoration so grouped content is easy to scan. Structure, spacing, and typography should carry most of the organization instead of a custom visual identity.
