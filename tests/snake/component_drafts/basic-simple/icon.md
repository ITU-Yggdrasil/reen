# Icon

## Description
Icon is a compact visual symbol used to support recognition, scanning, and quick comprehension. Use it to reinforce meaning alongside text or to represent common actions and statuses, and avoid relying on icon-only communication when text is necessary for clarity.

## Purpose
The purpose of Icon is to reinforce meaning, improve scanning, and support quick recognition.

## Visual Structure
Icon is a simple, clean symbol with consistent visual weight and bounded size behavior. It should align neatly with adjacent text or controls and remain legible at small scales without unnecessary detail.

## Subcomponents
None.

## States & Variants
- States: default, hover-linked, active, disabled, status-colored where documented.
- Variants: outline, filled, navigation, action, status.
- Sizes: small, medium, large.

## Properties
- `name`: icon identifier.
- `size`: rendered dimensions.
- `decorative`: whether assistive technologies should ignore it.
- `label`: accessible name when the icon carries meaning.
- `tone`: neutral or semantic emphasis.
- `placement`: standalone, leading, trailing, or inline.

## Brand Reference
No formal brand specification is required for this draft. Icon should use simple shapes, consistent visual weight, and clear legibility at small sizes. Any color or emphasis should support meaning, not ornament, and icons should work comfortably inside a neutral interface system.
