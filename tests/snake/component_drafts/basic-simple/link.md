# Link

## Description
Link is a navigational text component that moves the user to another destination, view, section, or external resource. Use it for navigation and reference actions, and avoid styling it like a primary action button unless the destination behavior truly needs button treatment.

## Purpose
The purpose of Link is to connect users to related destinations, pages, sections, or resources.

## Visual Structure
The link is typically inline or lightly separated from surrounding content, with a readable text label and optional supporting icon. The form should remain lighter than a Button while still being easy to scan and activate.

## Subcomponents
None.

## States & Variants
- States: default, hover, focus-visible, active, visited, disabled when applicable.
- Variants: inline, standalone, subdued, prominent.
- Icon positions: leading or trailing.

## Properties
- `label`: visible link text.
- `href` or `destination`: target location.
- `external`: indicates a destination outside the current experience.
- `iconLeading`: optional icon before the text.
- `iconTrailing`: optional icon after the text.
- `variant`: controls visual prominence.
- `visited`: optional documented state where applicable.

## Brand Reference
No formal brand specification is required for this draft. Link should remain readable, clearly interactive, and visually lighter than stronger action components. Styling should prioritize contrast, discoverability, and consistency without relying on custom brand colors or decorative effects.
